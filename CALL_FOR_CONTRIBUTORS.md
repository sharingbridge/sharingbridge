# Join Us in Building ShareBridge - A Platform for Dignified Giving

## 🌟 Project Overview

**ShareBridge** is an open-source, AI-driven initiative to transform charitable giving by creating a transparent, dignified platform that connects donors with alms seekers through verified food delivery services. We ensure charitable donations reach their intended purpose - providing food and essentials rather than cash that could be misused.

**Our Mission:** Enable compassionate giving while maintaining dignity for all parties involved.

## 🤖 Development Approach: AI-Powered Collaboration

**ShareBridge is being built through AI-assisted development!** Most code, documentation, and artifacts are generated through carefully crafted prompts to AI systems. This means:

- ✨ **You don't need to be a coding expert** to contribute meaningfully
- 📝 **Contribution happens through prompting** - describing what you need in natural language
- 🗂️ **All prompts are stored** in the `PROMPTS.md` file and in each repository's `prompting/` folder
- 🤝 **Diverse expertise matters more than technical skills** - we need humanitarian perspectives, legal guidance, and community insights
- 💡 **Discussion-driven development** - share ideas in GitHub Discussions, and the community refines them into prompts

**This changes everything.** Whether you're a social worker, lawyer, community organizer, or concerned citizen - your domain knowledge is just as valuable as coding skills.

---

## 🤝 Why This Matters

Every day, people encounter those seeking alms and face a difficult choice - help with cash that may be misused, or walk away. ShareBridge solves this by:

- ✅ Ensuring donations become meals and essentials, not cash
- ✅ Providing delivery confirmation with photo proof
- ✅ Maintaining dignity for both donors and seekers
- ✅ Leveraging existing delivery infrastructure
- ✅ Creating community solidarity through pledge pools and crowdfunding

**Impact Potential:** Millions of charitable interactions yearly could become transparent, accountable acts of kindness.

## 📁 Repository Structure & Prompting Approach

**Independent Evolution. Coordinated Progress. AI-Powered Development.**

ShareBridge uses a **multi-repo architecture** where each stream evolves independently, with development driven primarily through **AI-assisted prompting**.

### Master Repository (This Repo)
- **Purpose:** Overall coordination, documentation, requirements
- **Contains:** Business requirements, architecture docs, progress tracking, **master PROMPTS.md file**
- **Track:** Links to all child repositories and their status

### Child Repositories (Independent Streams)

Each repository contains:
- Source code and artifacts
- **`prompting/` folder** - All prompts used to generate code, docs, and configs for that specific component
- Issues and discussions specific to that component
- Independent CI/CD pipelines

**Frontend Repositories:**
- `ShareBridge-mobile-app` - React Native/Flutter mobile application
  - `prompting/` - Mobile UI/UX prompts, feature requests
- `ShareBridge-web-app` - React/Next.js web application
  - `prompting/` - Web interface prompts, accessibility requirements

**Backend Repositories:**
- `ShareBridge-api-gateway` - API gateway and routing
  - `prompting/` - API design prompts, routing logic
- `ShareBridge-order-service` - Order management microservice
  - `prompting/` - Order flow prompts, business logic
- `ShareBridge-user-service` - User authentication and profile service
  - `prompting/` - Auth flow prompts, user management
- `ShareBridge-integration-service` - Vendor integration (Swiggy, Zomato, etc.)
  - `prompting/` - Integration prompts, vendor API specs
- `ShareBridge-notification-service` - Push notifications and alerts
  - `prompting/` - Notification logic prompts

**AI/ML Repositories:**
- `ShareBridge-ai-safety` - Location safety assessment models
  - `prompting/` - AI model prompts, training data descriptions
- `ShareBridge-photo-service` - Face detection and beneficiary assistance history matching
  - `prompting/` - Computer vision prompts, privacy requirements

**Infrastructure Repositories:**
- `ShareBridge-infra` - Infrastructure as Code (Terraform/CloudFormation)
  - `prompting/` - Infrastructure prompts, deployment specs
- `ShareBridge-deployment` - CI/CD pipelines and deployment scripts
  - `prompting/` - Pipeline prompts, deployment strategies

**Note:** Documentation is maintained within each service repository rather than in a separate docs repo. User guides go in mobile/web repos, API docs in service repos, deployment guides in infra repos.

### Why Prompting Folders?

**Transparency:** Anyone can see the prompts that generated code/docs  
**Reproducibility:** Others can regenerate or modify artifacts using similar prompts  
**Learning:** New contributors learn by seeing effective prompts  
**Collaboration:** Non-coders contribute by refining prompts  
**AI Evolution:** As AI tools improve, we can regenerate components with better results
