# AI orchestration — local Python (optional)

**Repository:** `sharingbridge-ai-orchestration`  
**Not part of this `configuration/` folder.** Nothing here is committed: no `.venv`, no `Activate.ps1`, no database rows.

## When you need it

| Testing | Need this service? |
|---------|-------------------|
| Web dashboard, mobile donor, Postgres presets, order intents | **No** — only user-service (`8081`) + integration-service (`8080`) |
| Query-ranked **Suggest Vendors** or live **instruction-pack** text | **Yes** — integration-service calls this service on port **8091** |

Without it, integration-service uses built-in mocks (`AI_*_ENABLED` off or orchestration unreachable).

## What is uvicorn?

**Uvicorn** is the Python HTTP server that runs the FastAPI app in `sharingbridge-ai-orchestration`. It is the Python equivalent of `npm start` for the Node services.

```text
integration-service :8080  ──HTTP──►  uvicorn :8091  ──►  FastAPI (suggest-vendors, instruction-pack)
```

Mobile and web **never** call uvicorn directly.

## Virtualenv and `Activate.ps1` (Windows)

Create the venv **only inside the AI repo**, not under a parent folder like `D:\kannan\sharingbridge\.venv`:

Requires **Python 3.10+** (3.13 works). Use `python3.13` — not Anaconda’s old `python` (often 3.7), which breaks FastAPI/Pydantic.

```powershell
cd D:\kannan\sharingbridge\sharingbridge-ai-orchestration
python3.13 -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
$env:PORT = "8091"
uvicorn app.main:app --host 0.0.0.0 --port 8091
```

| File | In git? | Purpose |
|------|---------|---------|
| `configuration/*.sql`, `*.md` | Yes | Schema, coordinator seed SQL, env wiring docs |
| `sharingbridge-ai-orchestration/.venv/` | **No** (gitignored) | Your machine’s Python packages |
| `.venv\Scripts\Activate.ps1` | **No** | Generated by `python3.13 -m venv`; activates that venv in PowerShell |

If `Activate.ps1` is **empty** or activation fails, delete `.venv` in that repo and run `python3.13 -m venv .venv` again. If `uvicorn` crashes with `ForwardRef._evaluate() … recursive_guard`, recreate the venv and reinstall (Pydantic v1 is incompatible with Python 3.13). Alternative without activation:

```powershell
.\.venv\Scripts\python.exe -m uvicorn app.main:app --host 0.0.0.0 --port 8091
```

## Wire integration-service

In `sharingbridge-integration-service/.env`:

```env
AI_ORCHESTRATION_BASE_URL=http://localhost:8091
AI_SUGGEST_VENDORS_ENABLED=true
AI_INSTRUCTION_PACK_ENABLED=true
```

Restart integration-service after changing env. See [MANUAL_TESTING_GUIDE.md](../testing/MANUAL_TESTING_GUIDE.md) §1d and §2.
